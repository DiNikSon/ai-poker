## Правила Texas Hold'em

### Цель игры

Выиграть фишки соперников, собрав лучшую комбинацию из пяти карт.

### Раунды торговли

1. **Pre-flop** - После раздачи двух закрытых карт каждому игроку
2. **Flop** - После открытия первых трёх общих карт
3. **Turn** - После открытия четвёртой общей карты
4. **River** - После открытия пятой общей карты
5. **Showdown** - Вскрытие карт (если игроки дошли сюда)

### Возможные действия в ваш ход

- **Check** (чек) - Остаться в игре без ставки (доступно только если никто не ставил)
- **Bet** (ставка) - Сделать первую ставку
- **Call** (уравнять) - Уравнять текущую ставку
- **Raise** (рейз) - Увеличить ставку
- **Fold** (сброс) - Выйти из раунда (потеряете вложенные фишки)
- **All-in** (олл-ин) - Поставить все оставшиеся фишки

### Комбинации (старшинство от высокого к низкому)

| Комбинация      | Описание                     | Пример         |
| --------------- | ---------------------------- | -------------- |
| Straight Flush  | Пять карт подряд одной масти | ♥9 ♥8 ♥7 ♥6 ♥5 |
| Four of a Kind  | Четыре карты одного ранга    | ♠K ♥K ♣K ♦K ♣7 |
| Full House      | Тройка + пара                | ♠7 ♥7 ♣7 ♠2 ♥2 |
| Flush           | Пять карт одной масти        | ♣2 ♣4 ♣6 ♣8 ♣T |
| Straight        | Пять карт подряд             | ♠5 ♥6 ♣7 ♦8 ♠9 |
| Three of a Kind | Тройка карт одного ранга     | ♠J ♥J ♣J ♦2 ♣3 |
| Two Pair        | Две пары                     | ♠4 ♥4 ♣8 ♦8 ♠K |
| One Pair        | Одна пара                    | ♠9 ♥9 ♣2 ♦5 ♣7 |
| High Card       | Старшая карта                | ♠A ♥K ♣Q ♦J ♣9 |

### Структура игры

- **Small blind** - обязательная малая ставка (игрок слева от дилера)
- **Big blind** - обязательная большая ставка (игрок слева от small blind)
- **Минимум 2 игрока**, максимум 8 игроков за столом
- **Раунд завершается**, когда все игроки либо выбыли, либо уравняли ставку

### Повышение блайндов

Блайнды увеличиваются каждые 10 раздач:

| Раздачи | Small Blind | Big Blind |
|---------|-------------|-----------|
| 1–10    | 1           | 2         |
| 11–20   | 2           | 4         |
| 21–30   | 3           | 6         |
| 31–40   | 5           | 10        |
| 41–50   | 10          | 20        |
| 51–60   | 15          | 30        |
| 61–70   | 20          | 40        |
| 71+     | 40          | 80        |

Учитывайте это при построении стратегии: агрессивная игра становится выгоднее по мере роста блайндов.

## Структура бота

Ваш файл `bot.js` должен содержать объект с двумя частями:

### 1. Конфигурация (config)

```javascript
  config: {
    name: "Имя вашего бота",
    image_url: "https://example.com/avatar.jpg",
    author_name: "Иван Иванов",
    description: "Краткое описание бота...",
    win_say: [
      "Я - король покера! 👑",
      "Неплохо для второго места",
      "Третье место - не худший результат"
    ],
    lose_say: "В следующий раз повезёт 🍀"
  },
```

**Параметры:**

| Параметр      | Тип    | Описание                  | Лимит                         |
| ------------- | ------ | ------------------------- | ----------------------------- |
| `name`        | String | Отображаемое имя бота     | max 50 символов               |
| `image_url`   | String | URL изображения аватара   | Должен быть доступен по HTTPS |
| `author_name` | String | Ваше имя или никнейм      | max 100 символов              |
| `description` | String | Описание бота или приветствие  | max 500 символов              |
| `win_say[0]`  | String | Победная фраза за 1 место | max 100 символов              |
| `win_say[1]`  | String | Фраза за 2 место          | max 100 символов              |
| `win_say[2]`  | String | Фраза за 3 место          | max 100 символов              |
| `lose_say`    | String | Фраза при выбытии         | max 100 символов              |

### 2. Функция turn

```javascript
turn: ({ game, memory }) => {
  game.fold();
  return memory;
}
```

**Параметры функции:**

- **game** - Объект с методами для совершения действий и текущее состояние
- **memory** - Объект для сохранения состояния между ходами в течение одной игры
- **Возвращаемое значение:** Обновленный объект `memory` (если вернуть `null`, `undefined` или не вернуть ничего вовсе, объект памяти останется таким же)

**Требования и ограничения:**

- Функция должна быть **синхронной** — `async/await`, `Promise`, `setTimeout` и другие асинхронные конструкции не поддерживаются
- Запрещено обращаться к **`localStorage`**, **`sessionStorage`** и любому другому хранилищу браузера — используйте `memory` для сохранения данных между ходами
- Запрещено обращаться к **DOM** (`document`, `window`, `querySelector` и т.д.)
- Запрещены сетевые запросы (`fetch`, `XMLHttpRequest` и т.д.)

**Использование ИИ:**

ИИ можно использовать как помощника при написании кода — для отладки, синтаксиса, реализации алгоритмов. Но стратегию должны придумывать **вы сами**: если попросить ИИ разработать стратегию целиком, все боты окажутся одинаковыми и соревнование потеряет смысл. Ваша идея — ваше преимущество.

## API

### Объект game

#### game.fold()

Выйти из раунда, потеряв все вложенные фишки в этот раунд.

#### game.check()

Остаться в игре без ставки (доступно только если никто не повышал ставку).

#### game.bet(amount: int)

Основной метод ставки. Добавляет указанное количество фишек к вашей текущей ставке.

#### game.call()

Уравнять текущую ставку соперников. Сокращение для `game.bet(game.to_beat)`.

#### game.raise(amount: int)

Увеличить ставку на указанную сумму поверх текущей максимальной ставки. Сокращение для `game.bet(game.to_beat + amount)`.

#### game.all_in()

Поставить все оставшиеся фишки. (Даже если не хватает до текущей ставки соперника, в таком случае вы больше не участвуете в ставках и при выигрыше получите только сумму эквивалентную вашей ставке)

**Если вы вызовете метод действия дважды или больше за один ход, будет выполнено последнее.**

### Обработка некорректных действий

Контроллер автоматически заменяет невозможные действия на ближайшие допустимые:

| Вызов | Ситуация | Фактическое действие |
|-------|----------|---------------------|
| `fold()` | - | Fold (доступно всегда) |
| `check()` | Кто-то уже поставил (`to_beat > current_bet`) | Fold |
| `bet(0)` | - | Check |
| `bet(amount)` | Не хватает фишек (`amount > funds`) | All-in |
| `call()` | - | Аналогично bet(`to_beat`) |
| `raise(amount)` | - | Аналогично bet(`to_beat + amount`) |
| `all_in()` | - | All-in (доступно всегда) |

#### game.say(message: string)

Вывести сообщение в чат игры. Это чисто визуальное сопровождение — текст отобразится рядом с вашим ботом на столе, но никак не влияет на ход игры. Если вызвать `say()` несколько раз за один ход, отобразится только последнее сообщение.

#### game.current_bet: int

Значение вашей текущей ставки

#### game.funds: int

Значение доступных для ставки фишек

#### game.pot: int

Общий банк текущего раунда — сумма всех ставок всех игроков. Это фишки, которые заберёт победитель (или победители, если банк делится). Используйте для расчёта пот-оддсов: соотношения размера банка к стоимости вашего колла.

#### game.to_beat: int

Количество фишек, которое нужно добавить к вашей ставке, чтобы уравнять текущую максимальную ставку за столом. Если `to_beat` равен 0, можно сделать чек.

#### game.hand: Array<Array(2)>(2)

Список карт у вас в руке в формате:
```javascript
[
    [1, "d"],
    [0, "c"],
]
```
Где первый элемент - номинал карты (0 - "2", 12 - "Туз"), 

второй элемент - масть
"c" - clubs - трефы
"s" - spades - пики
"d" - diamonds - бубны
"h" - hearts - червы

#### game.community: Array<Array(2)>(0..5)

Список общих карт на столе в формате, описанном выше, размер массива: 0, 3, 4 или 5 карт в зависимости от этапа раунда.

#### game.players: Array\<Object>

Список всех игроков за столом. Каждый элемент содержит:
```javascript
{
  position: 0,
  funds: 950,
  current_bet: 50,
  is_folded: false,
  is_all_in: false
}
```

| Поле          | Тип   | Описание                                    |
| ------------- | ----- | ------------------------------------------- |
| `position`    | int   | Позиция игрока за столом (начиная с 0)      |
| `funds`       | int   | Оставшиеся фишки игрока                     |
| `current_bet` | int   | Текущая ставка игрока в этом раунде торговли |
| `is_folded`   | bool  | Сбросил ли игрок карты                       |
| `is_all_in`   | bool  | Пошёл ли игрок ва-банк                      |

#### game.player_count: int

Количество игроков за столом в текущем раунде (включая выбывших по fold в этом раунде).

#### game.position: int

Позиция вашего бота за столом (начиная с 0).

#### game.first_player: int

Позиция игрока, который ходит первым в текущем раунде (small blind). В начале каждого раунда (Префлоп) этот игрок принудительно ставит **1 фишку** (малый блайнд), а следующий за ним — **2 фишки** (большой блайнд), с последующим увеличением. Эти ставки обязательны и делаются автоматически до начала торговли.

#### game.round: int

Номер раунда (начиная с 0)

#### game.step: int

Номер этапа раунда:
- 0 - Префлоп - начальные ставки до раздачи общих карт (малый блайнд: 1 фишка, большой блайнд: 2 фишки)
- 1 - Флоп - выложено 3 общих карты
- 2 - Тёрн - выложено 4 общих карты
- 3 - Ривер - выложены все 5 общих карт, финальные ставки до определения победителя

### Объект Memory
Объект для сохранения информации между ходами:
```javascript
turn: ({ game, memory }) => {
  let stats = memory ?? { wins: 0, losses: 0 }
  stats.wins++
  return stats
}
```

**Ограничения:**
- Максимум 10 KB данных
- Сохраняется между раундами одной игры
- Сбрасывается при начале новой игры

## Примеры
### Пример: Простой агрессивный бот
```javascript
{
  config: {
    name: "Агрессор",
    image_url: "https://fastly.picsum.photos/id/237/200/200.jpg",
    author_name: "Иван Иванов",
    description: "Агрессивная стратегия: часто повышает ставки",
    win_say: ["Агрессия победила! 💪", "На следующий раз буду ещё агрессивнее", "И третьего раза не будет"],
    lose_say: "Иногда агрессия не помогает"
  },
  turn: ({ game, memory }) => {
    if (game.hand.some(card => card[0] === 12)) {
      game.raise(100)
    } else {
      game.call()
    }
    return null
  }
}
```

| Минимальная ставка | 1 фишка | |
